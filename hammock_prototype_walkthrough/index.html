Shell prototype walk-through

  • start webserver in /dist
  • open index.html
  • you'll see:

1. First, let's create a class

  • click classes
  • click New
  • you'll see a new object, with an auto-generated _id
  • replace the auto-generated _id with "user"
  • set title field to "Users"
  • click Save
  • the browser will ask you if you want to create a new object, click Yes
  • click classes again, to refresh middle column (this is bug for now)
  • congrats, you've created your first class!

2. Now you can create a user!

  • Users in left nav
  • click New
  • you'll see autogenerated _id and class fields. You can leave this.
  • set title_field (this will be the user name)
  • set body_field (this will be the users full name)
  • click Users again, to refresh middle column (this is bug for now)

3. Create Posts class

  • click classes, then New
  • set _id to "post"
  • set title field to "Posts"
  • Save

4. Create a post

  • click Posts in left nav
  • click New
  • you'll see autogenerated _id and class fields
  • set title_field (this will be the post title)
  • set body_field (this will be the post content)
  • click Posts again, to refresh middle column (this is bug for now)

Congrats, you've created Users and Posts with the Hammock Shell!
